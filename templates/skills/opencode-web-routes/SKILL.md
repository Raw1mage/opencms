# Opencode Gateway: Web Route Registration

The opencode C gateway daemon (listens on `:1080`) routes path-prefixed HTTP requests to local upstream services. There are **two files** to understand, and only one of them should be edited directly.

## Two-Layer Architecture

| File | Owner | Purpose | Edit? |
|---|---|---|---|
| `~/.config/web_registry.json` | per-user (XDG_CONFIG_HOME) | Declarative registry of this user's webapps. Read by opencode server for `/api/v2/web-route/health` (TCP probe) and `/toggle` (invokes `webctl.sh start/stop`). | **Yes — edit directly.** |
| `/etc/opencode/web_routes.conf` | gateway daemon | Runtime routing table. Format: `<prefix> <host> <port> <owner_uid> [auth]`. Header says "do not edit manually" — it's flushed by the gateway. | **No — go through ctl.sock.** |

The two layers are independent: editing the registry does NOT create a route; calling `publish` does NOT add to the registry. For a new webapp you need to do **both**.

## Key Runtime Endpoints

- **Gateway control socket**: `/run/opencode-gateway/ctl.sock` (newline-delimited JSON, one response per request)
- **Gateway HTTP port**: `1080` (configurable via `OPENCODE_GATEWAY_PORT`)
- **Config dir**: `/etc/opencode/`
- **Gateway source**: `/home/pkcs12/projects/opencode/daemon/opencode-gateway.c`
- **Registry code**: `/home/pkcs12/projects/opencode/packages/opencode/src/server/routes/web-route.ts`

## web_routes.conf Format

```
# Auto-generated by opencode-gateway — do not edit manually
# Format: <prefix> <host> <port> <owner_uid> [auth]
# auth: 0=public (default), 1=require login
/cecelearn/api 127.0.0.1 3014 1000 0
/cecelearn     127.0.0.1 5173 1000 0
/linebot/api/webhook 127.0.0.1 3015 1000 0
/linebot/api   127.0.0.1 3015 1000 1
/linebot       127.0.0.1 3015 1000 1
```

- Longest prefix wins (gateway sorts by prefix length descending on load).
- `auth=0` → public; `auth=1` → JWT login required (except excluded sub-paths like webhook).
- `owner_uid` filters which user sees the route in `GET /api/v2/web-route/`.

## web_registry.json Schema

```json
{
  "version": 1,
  "entries": [
    {
      "entryName": "cisopro",
      "projectRoot": "/home/pkcs12/projects/cisopro",
      "publicBasePath": "/cisopro",
      "host": "127.0.0.1",
      "primaryPort": 5150,
      "webctlPath": "/home/pkcs12/projects/cisopro/webctl.sh",
      "enabled": true,
      "access": "protected"
    }
  ]
}
```

Field notes:
- `access` is a registry-level label (`"public"` / `"protected"`); it does **not** set the gateway auth flag by itself — you still pass `auth` in the publish payload.
- `webctlPath` must exist and accept `start` / `stop` subcommands (the `/toggle` endpoint spawns it).

## Publishing a Route (Canonical Flow)

### Option A — via opencode source repo's webctl.sh (recommended)

```bash
cd /home/pkcs12/projects/opencode
./webctl.sh publish-route /cisopro 127.0.0.1 5150
./webctl.sh list-routes
./webctl.sh remove-route /cisopro        # to undo
```

This wraps the ctl.sock JSON protocol. Limitation: does **not** expose the `auth` field.

### Option B — direct ctl.sock call (full control, supports auth)

```bash
echo '{"action":"publish","prefix":"/cisopro","host":"127.0.0.1","port":5150,"auth":1}' \
  | socat - UNIX-CONNECT:/run/opencode-gateway/ctl.sock
```

Other actions: `{"action":"list"}`, `{"action":"remove","prefix":"/cisopro"}`.

### Option C — opencode HTTP API (from the opencode web UI context)

```
POST /api/v2/web-route/publish    {prefix, host, port}
POST /api/v2/web-route/remove     {prefix}
GET  /api/v2/web-route/           → user-scoped list (filtered by process uid)
GET  /api/v2/web-route/health     → TCP-probe every registry entry
POST /api/v2/web-route/toggle     {entryName, action: "start"|"stop"}
```

The HTTP API does not currently forward `auth` either — for login-required routes, use Option B once, then the route sticks (persisted in `web_routes.conf`).

## Full Procedure for a New Webapp

1. **Pick a prefix and port.** Prefix must start with `/` and be unique. Port must not collide with existing entries (check `ss -tlnp | grep LISTEN`).
2. **Append to `~/.config/web_registry.json`** — add an entry object to `entries[]`. This is a plain JSON edit; no tool needed.
3. **Ensure the upstream service is running** on `127.0.0.1:<port>`. The gateway does not start services for you — `toggle` does, but the webctl.sh must already support `start`.
4. **Publish via ctl.sock**:
   - Public (no login): `{"action":"publish","prefix":"/foo","host":"127.0.0.1","port":N}`
   - Protected: add `"auth":1`
5. **Verify**: `cat /etc/opencode/web_routes.conf` shows the new line; `curl http://localhost:1080/foo/` returns the upstream response (or 401 if protected and no JWT).

## Troubleshooting Cheat Sheet

| Symptom | Likely cause |
|---|---|
| `gateway unreachable` from ctl.sock | Gateway daemon not running. Check `pgrep -af opencode-gateway`; restart via `webctl.sh gateway-start` in the opencode source repo. |
| Publish succeeds but requests return 502/504 | Upstream not listening on the declared port. `ss -tlnp \| grep :<port>`. |
| Route returns 401 unexpectedly | `auth=1` was set. Either re-publish without auth, or login via `/auth/login` to get a JWT cookie. |
| Route missing from opencode web UI sidebar | The UID filter in `GET /api/v2/web-route/` drops routes owned by other users. Confirm the uid column in `web_routes.conf` matches `id -u`. |
| Health endpoint shows `alive: false` | TCP probe to `primaryPort` failed. Service not running, or firewalled. |
| Edits to `web_routes.conf` disappear | The gateway re-flushes on every publish/remove. Never edit the file directly; if you need a multi-line entry (e.g., webhook with `auth=0`, main path with `auth=1`), call publish once per line. |

## Do NOT

- Do **not** hand-edit `/etc/opencode/web_routes.conf` — the gateway overwrites it on the next publish/remove. (Yes, `linebotweb/webctl.sh` does this via `sudo tee -a`; it works until someone calls publish.)
- Do **not** omit the `~/.config/web_registry.json` entry — the route will work but won't show in the opencode web UI's managed-apps list, and `/toggle` / `/health` won't know about it.
- Do **not** assume `webctl.sh publish-route` sets auth — it does not. For login-required routes, hit `ctl.sock` directly.
