- **Summary**: Resolved the issue where `bun turbo typecheck` hung indefinitely due to a background `setInterval` in the autonomous supervisor. This manifested as a `SIGHUP` (or hang) during the `@opencode-ai/sdk:build` process.
- **Debug Checkpoints**:
    - **Baseline**:
        - Symptoms: `bun turbo typecheck` hangs at `@opencode-ai/sdk:build`.
        - Trace: `strace` showed the Bun process waiting on `epoll_pwait` while an interval timer was active.
    - **Root Cause**:
        - `packages/opencode/src/openapi/generate.ts` calls `Server.openapi()`, which initializes the full Hono app.
        - `createApp()` accidentally starts the `ensureAutonomousSupervisor()` interval, preventing the script from exiting after spec generation.
    - **Execution**:
        - Added `process.exit(0)` to `packages/opencode/src/openapi/generate.ts` after successful spec write.
        - Replaced `$` calls with `execSync` in `packages/sdk/js/script/build.ts` for more robust process management.

        - Individual package checks: All clean.
